NIVEL 1 — BÁSICO
¿Qué es el Ladder Diagram?
El lenguaje de programación industrial más extendido, nacido del cableado eléctrico.
Origen del Ladder
DEL RELÉ AL SOFTWARE

El Ladder (escalera) nació como representación gráfica del cableado de armarios con relés electromagnéticos. Los ingenieros necesitaban documentar los circuitos de control y decidieron dibujarlo como si fuera un circuito eléctrico visto desde arriba: dos raíles verticales (L1 y L2) con contactos y bobinas en horizontal, como los peldaños de una escalera.

Cuando llegaron los PLCs (Controladores Lógicos Programables) en los años 70, se adoptó este mismo lenguaje para que los electricistas pudieran programar sin necesidad de aprender lenguajes de texto. Hoy sigue siendo el lenguaje más usado en automatización industrial.

ESTRUCTURA BÁSICA

L1 (barra izquierda): Siempre energizada. Es el polo positivo / fase del sistema.

L2 (barra derecha): Neutro o retorno. La corriente "busca" llegar aquí.

Rung: Cada línea horizontal entre L1 y L2. Contiene la lógica de una condición→acción.

Scan: El PLC evalúa TODOS los rungs de arriba a abajo, una y otra vez, cíclicamente.

CÓMO LEER UN RUNG

Lee cada rung como una frase lógica:

"SI [contacto A] Y [contacto B] ENTONCES activar [bobina C]"

Los contactos a la izquierda son las condiciones. La bobina a la derecha es la acción. La corriente fluye de izquierda a derecha si todas las condiciones se cumplen.

💡 El PLC no piensa: solo evalúa la lógica que tú programas. Si la lógica está bien, el control funciona.

NIVEL 1 — BÁSICO
Contacto NA — Normalmente Abierto
La instrucción de entrada más común: cierra el circuito cuando su variable vale 1.
—| |—
NO Contact / XIC (Examine If Closed)
IEC 61131-3 · CONTACTO NORMALMENTE ABIERTO
DEFINICIÓN

Conduce corriente solo cuando su variable (tag) vale 1 (TRUE). Si el tag es 0, el contacto está abierto y no pasa corriente.

TABLA DE VERDAD
Tag (variable)ContactoCorriente
0 (FALSE)AbiertoNO pasa
1 (TRUE)CerradoSÍ pasa

En el mundo físico equivale a:

• Pulsador de marcha (normalmente abierto)

• Sensor inductivo activo en alto

• Final de carrera de confirmación

EJEMPLO PRÁCTICO

Rung: ──[PB_MARCHA]──[PROTECCION_OK]──(MOTOR)──

El motor arranca solo si el pulsador de marcha está pulsado (PB_MARCHA = 1) Y la protección no ha disparado (PROTECCION_OK = 1). Cualquier condición en 0 abre el circuito.

💡 Naming: usa nombres descriptivos como PB_MARCHA, SENSOR_FIN, NIVEL_MAX. Facilita el mantenimiento y el diagnóstico de averías.
▷ Practicar: colocar un NO Contact
NIVEL 1 — BÁSICO
Contacto NC — Normalmente Cerrado
Conduce por defecto y se abre cuando su variable se activa. Esencial en lógica de seguridad.
—|/|—
NC Contact / XIO (Examine If Open)
IEC 61131-3 · CONTACTO NORMALMENTE CERRADO
DEFINICIÓN

Conduce corriente cuando su variable vale 0 (FALSE). Se ABRE (corta) cuando el tag = 1. Lógica inversa.

TABLA DE VERDAD
Tag (variable)ContactoCorriente
0 (FALSE)CerradoSÍ pasa
1 (TRUE)AbiertoNO pasa

En el mundo físico equivale a:

• Seta de emergencia (abre cuando se pulsa)

• Protección térmica de motor (abre al disparar)

• Final de carrera de límite de seguridad

PRINCIPIO FAIL-SAFE

Los dispositivos de seguridad se cablean NC en la realidad. Así, si el cable se rompe o hay un fallo, la señal pasa a 0 → el contacto NC abre → la máquina para. El fallo siempre provoca la parada, no el arranque.

⚠️ Error común: cablear las setas de emergencia como NO. Si el cable se rompe, la seta "no funciona" pero no para nada. Siempre NC para seguridad.
▷ Practicar: crear lógica de paro de emergencia
NIVEL 1 — BÁSICO
Bobina de Salida — COIL
La instrucción de salida básica. Escribe el resultado del rung en una variable o activa una salida física.
—( )—
Output Coil / OTE (Output Energize)
IEC 61131-3 · BOBINA DE SALIDA
DEFINICIÓN

Escribe el estado del rung en su variable. Si el rung conduce → tag = 1. Si no conduce → tag = 0. Se evalúa en cada ciclo de scan.

Siempre va al final (derecha) del rung. Nunca en el medio.

COMPORTAMIENTO
RungBobina (tag)
Conduce (corriente)Tag = 1 (ON)
No conduceTag = 0 (OFF)
⚠️ No uses el mismo tag en dos bobinas COIL. El segundo rung sobrescribirá al primero en cada scan. Usa SET/RESET si necesitas controlar desde varios puntos.
▷ Practicar: rung básico NO → COIL

NIVEL 2 — COMBINATORIA
Enclavamiento — SET y RESET
Memorizar el estado de una salida aunque la condición de arranque desaparezca. El equivalente digital del relé de enclavamiento.
─(S)─ / ─(R)─
Set Coil + Reset Coil — Latch / Unlatch
ENCLAVAMIENTO CONTROLADO POR DOS CONDICIONES
CONCEPTO

El enclavamiento es el circuito de control más importante de la automatización. Permite arrancar un motor con un pulso breve (pulsador de marcha) y mantenerlo encendido aunque se suelte el pulsador.

En Ladder se implementa con SET y RESET:

Rung 1: ──[PB_MARCHA]──(S: MOTOR)── ← Un pulso enciende

Rung 2: ──[PB_PARO]────(R: MOTOR)── ← Un pulso apaga

MOTOR queda en 1 hasta que se active PB_PARO, aunque PB_MARCHA ya no esté pulsado.

COMPORTAMIENTO SET
Rung SETVariable
Conduce (pulso)Tag = 1 (queda)
No conduceTag = 1 (mantiene)
COMPORTAMIENTO RESET
Rung RSTVariable
Conduce (pulso)Tag = 0 (queda)
No conduceTag = 0 (mantiene)
⚠️ Prioridad RST: si SET y RST del mismo tag se evalúan en el mismo ciclo de scan, el RST gana (se evalúa después). Diseña siempre para que no ocurran a la vez.
▷ Practicar: circuito de enclavamiento
NIVEL 2 — COMBINATORIA
Detección de Flancos — P y N
Detectar el instante exacto de cambio de una señal, no su estado sostenido.
|P| / |N|
Rising Edge (P) y Falling Edge (N)
UN SOLO CICLO DE SCAN DE DURACIÓN
FLANCO POSITIVO — P

Activo durante un solo ciclo de scan en el flanco 0→1 de la señal. Independiente de cuánto dure la señal en 1.

CUÁNDO USAR CADA UNO

Flanco P (subida):

• Registrar el primer pulso de un pulsador

• Incrementar contador sin rebotes

• Disparar acción puntual al activar sensor

Flanco N (bajada):

• Registrar suelta de pulsador

• Detectar fin de movimiento

• Cambiar paso de secuencia al terminar

💡 Los flancos no mantienen la salida. Combínalos con SET para acciones persistentes: el flanco P dispara el SET, y el resultado queda memorizado hasta el RST.
▷ Practicar: contador con flanco P
NIVEL 3 — TEMPORIZADORES
TON — Retardo de Conexión
Activar una salida después de que la condición lleva activa un tiempo determinado. El temporizador más usado en automatización.
□TON□
Timer On-Delay (TON)
IEC 61131-3 · BLOQUE DE FUNCIÓN
PINES Y PARÁMETROS
PinTipoFunción
INBOOL entradaHabilita el conteo
PTTIME presetTiempo hasta activar Q
QBOOL salidaTRUE cuando ET ≥ PT
ETTIME salidaTiempo transcurrido
CRONOGRAMA
CÓMO USARLO EN EL SIMULADOR

1. Coloca el TON como salida de un rung. La entrada del rung (contactos) conecta al pin IN del TON.

2. En otro rung, usa T1.Q como contacto NO. El contacto T1.Q cierra cuando el temporizador completa PT ms.

Rung 1: ──[PB_MARCHA]──(TON T1 PT=5000ms)──

Rung 2: ──[T1.Q]──(MOTOR_ARRANQUE)──

💡 Si IN cae a 0 antes de completar PT, el contador ET se reinicia a 0. El motor solo arranca si el pulsador se mantiene el tiempo suficiente.
▷ Practicar: temporizador TON con T1.Q
NIVEL 3 — TEMPORIZADORES
TOF — Retardo de Desconexión
Mantener una salida activa durante un tiempo después de que la condición de entrada desaparece.
□TOF□
Timer Off-Delay (TOF)
IEC 61131-3 · Q = 1 INMEDIATO, SE APAGA TRAS PT
DIFERENCIA CON TON
TONTOF

APLICACIONES TÍPICAS

• Ventilador de enfriamiento: sigue corriendo PT segundos después de parar el motor principal.

• Alumbrado escalonado: la luz sigue encendida PT ms después de salir de la zona.

• Electroválvula de purga: sigue abierta para vaciar el cilindro tras el paro de la prensa.

⚠️ Si reactivas IN durante el conteo de desconexión, el TOF cancela el conteo y Q vuelve a 1. El timer sólo arranca cuando IN está definitivamente en 0.
▷ Practicar: ventilador con TOF
NIVEL 3 — TEMPORIZADORES
TP — Pulso de Duración Fija
Genera un pulso de exactamente PT milisegundos a partir de cualquier señal, sin importar su duración.
□TP□
Timer Pulse (TP) — No retriggerable
IEC 61131-3 · PULSO DE DURACIÓN EXACTA
COMPORTAMIENTO

En el flanco de subida (0→1) de IN, Q = 1 durante exactamente PT ms. Aunque IN siga en 1 más tiempo, Q se apaga al llegar a PT. Si se activa IN de nuevo mientras Q sigue activo, se ignora.

Aplicaciones: timbre escolar, pulso de apertura de cerradura, señal de habilitación breve, marcado de inicio de ciclo.

💡 El TP garantiza que la duración del pulso de salida siempre sea exactamente PT, independientemente de la señal de entrada. Ideal cuando necesitas consistencia temporal.
▷ Practicar: pulso con TP

EJEMPLO: CONTEO DE PRODUCCIÓN

Rung 1: ──[SENSOR_PIEZA]──(CTU C1 PV=100)──

Rung 2: ──[C1.Q]──(ALARMA_LOTE)──

Rung 3: ──[PB_RESET]──(R: C1.Q)──

Cada pieza que pasa activa el sensor (flanco P). Al llegar a 100 piezas, C1.Q = 1 y activa la alarma de lote completo. El operario pulsa RESET para reiniciar.

⚠️ Diseña siempre el rung de reset. Sin él, el contador llega a PV y Q queda permanentemente en 1.
▷ Practicar: contar piezas con CTU

Almacén con 50 unidades. Cada salida decrementa CV. Q activa alarma de stock bajo.

NIVEL 4 — CONTADORES
CTUD — Contador Bidireccional
Puede contar hacia arriba y hacia abajo. Dos salidas: Q (lleno) y QD (vacío).
□CTUD□
Counter Up/Down (CTUD)
GESTIÓN DE ALMACÉN / BUFFER INDUSTRIAL
APLICACIÓN TÍPICA: BUFFER ENTRE MÁQUINAS

Sensor de entrada (CU) incrementa al entrar piezas. Sensor de salida (CD) decrementa al salir. Q = TRUE cuando buffer lleno (CV ≥ PV): para la máquina de entrada. QD = TRUE cuando vacío (CV ≤ 0): para la máquina de salida.

⚠️ Nunca actives CU y CD simultáneamente. Diseña la lógica con interlocks (un NC del otro) para que sean mutuamente excluyentes.
▷ Practicar: CTUD en buffer
AUTOMATISMOS TÍPICOS
Patrones de Automatización
Circuitos de control que aparecen repetidamente en la industria. Aprende el patrón, aplícalo a cualquier máquina.
01

Enclavamiento Básico — Motor ON/OFF

El circuito de arranque y paro más usado en la industria

Permite arrancar un motor con un pulso breve de marcha y pararlo con un pulso breve de paro, sin mantener pulsados los botones.

Rung 1: ──[PB_MARCHA]──[PB_PARO_NC]──[PROTECCION_NC]──(SET: MOTOR)──

Rung 2: ──[PB_PARO]──────────────────────────────────(RST: MOTOR)──

Rung 3: ──[MOTOR]──────────────────────────────────────(CONTACTOR_K1)──

  1. PB_MARCHA activa SET → MOTOR = 1 (queda memorizado)
  2. Se suelta PB_MARCHA → MOTOR sigue en 1 gracias al latch
  3. CONTACTOR_K1 = 1 → la bobina física cierra el circuito de potencia
  4. PB_PARO activa RST → MOTOR = 0 → contactor se desactiva
  5. PROTECCION dispara → NC abre → RST activa → motor para
02

Arranque Estrella-Triángulo

Reducir la corriente de arranque de motores grandes

El motor arranca en estrella (baja tensión) durante un tiempo fijo con TON, y luego conmuta a triángulo (plena tensión). Reduce la corriente de arranque al 33%.

  1. PB_MARCHA → SET: MARCHA_GENERAL
  2. MARCHA_GENERAL → activa CONTACTOR_ESTRELLA + temporizador TON (PT = 8 s)
  3. Al completarse TON → T1.Q = 1 → RST: CONTACTOR_ESTRELLA
  4. T1.Q activa CONTACTOR_TRIANGULO → motor en plena potencia
  5. Interlock: NC de ESTRELLA en el rung de TRIANGULO y viceversa
⚠️ Añade siempre un pequeño retardo entre la desconexión de ESTRELLA y la conexión de TRIANGULO para evitar cortocircuito de los contactores.
03

Semáforo de Peatones

Secuencia temporizada de estados con TON

Al pulsar el botón de paso de peatones, se inicia una secuencia: rojo de vehículos → verde de peatones (PT=15s) → rojo de peatones → verde de vehículos.

  1. PB_PEATON (flanco P) → SET: CICLO_ACTIVO
  2. CICLO_ACTIVO → ROJO_VEHICULOS = 1 + TON T1 (PT = 3 s)
  3. T1.Q → VERDE_PEATON = 1 + TON T2 (PT = 15 s)
  4. T2.Q → RST: VERDE_PEATON + ROJO_PEATON + TON T3 (PT = 3 s)
  5. T3.Q → RST: CICLO_ACTIVO → vuelve al estado inicial
04

Conteo de Producción con Alarma

CTU para gestionar lotes y avisar al operario

Contar piezas que salen de una línea. Al llegar al tamaño del lote, activar alarma y parar la línea. El operario retira el lote y pulsa RESET para continuar.

  1. SENSOR_PIEZA (flanco P) → CTU C1 (PV = 50)
  2. C1.Q → ALARMA_LOTE (luz + zumbador)
  3. C1.Q → RST: MOTOR_LINEA (para la cinta)
  4. Operario retira lote y pulsa PB_RESET
  5. PB_RESET → RST: C1.CV (contador a cero) + SET: MOTOR_LINEA
